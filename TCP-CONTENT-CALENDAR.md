## Guide Selection Strategy

### Category Rotation

Guides are organized into 5 pillars. Each week features guides from at least 3 different pillars to keep content diverse.

| Pillar | Example Guides | Priority Weight |
|---|---|---|
| **Getting Started** | eligibility-requirements, how-to-join, creativity-program-complete-guide-2026 | High (new audience acquisition) |
| **Maximize Earnings** | optimize-rpm, maximize-qualified-views, payout-timeline | High (core value prop) |
| **Growth** | tiktok-algorithm-2026, first-1000-followers, best-posting-schedule, tiktok-hook-formulas | High (broadest audience) |
| **Tools & Resources** | best-video-editing-apps, best-analytics-tools, best-scheduling-apps | Medium (affiliate potential) |
| **Advanced/Niche** | monetize-artists, q4-strategy, multiple-revenue-streams, creator-tax-guide | Medium (depth + authority) |

### Selection Rules

1. **Never repeat the same guide across platforms in the same week.** Each of the 7 selected guides maps to one primary platform slot.
2. **Rotate through pillars.** No two consecutive weeks should lead with the same pillar.
3. **Prioritize by SEO potential.** Guides targeting high-volume keywords (e.g., "TikTok Creator Rewards Program," "how to optimize TikTok RPM") get featured first.
4. **Seasonal/timely topics jump the queue.** Tax season = tax guide. Q4 = Q4 strategy. Algorithm update = algorithm guide.
5. **Track which guides have been featured.** Maintain a log at the bottom of this file. A guide can be re-featured after 8 weeks minimum.
6. **107 guides / 7 per week = 15+ weeks of unique content** before any guide needs repeating.

### Weekly Selection Process (Bernard)

Every Monday, Bernard:
1. Checks which pillars were used last week
2. Selects 7 guides from at least 3 different pillars
3. Assigns each guide to a platform slot
4. Routes to Scribe with the guide list + platform assignments
5. Routes to Vale with image briefs (1 per Instagram post + 1 per Pinterest pin = 12 images/week)
6. Reviews Scribe copy, reviews Vale images, then schedules via Zernio/Wren
